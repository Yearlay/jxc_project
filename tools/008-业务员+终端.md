# 业务员+终端

**功能**：实现三级菜单页面（parent_id=60，基础信息设置）
- 系统维护 → 基础信息设置 → 采购业务员（菜单 id=68，path=/system/basic/purchase-staff）
- 系统维护 → 基础信息设置 → 销售业务员（菜单 id=69，path=/system/basic/sales-staff）
- 系统维护 → 基础信息设置 → 销售终端（菜单 id=70，path=/system/basic/branch）

---
# 输入

### 采购业务员
1）字段说明：`name`（姓名）、`phone`（电话）、`address`（地址）、`is_default`（默认采购员：1=是，0=否）、`purchase_limit`（单次采购金额上限，数值，0表示不限）
2）默认数据：预写入一条 `name=店长`，`is_default=1`
3）业务说明：按姓名/电话关键词查询、新增、修改、删除
4）无状态字段
5）`is_default` 设为1时，自动将其他记录的 `is_default` 改为0（同一时间只允许一个默认）

### 销售业务员
1）字段说明：`name`（姓名）、`phone`（电话）、`address`（地址）、`is_default`（默认销售员：1=是，0=否）、`commission_rate`（销售提成比例，如 0.05 表示 5%）
2）默认数据：预写入一条 `name=店长`，`is_default=1`
3）业务说明：按姓名/电话关键词查询、新增、修改、删除
4）无状态字段
5）`is_default` 设为1时，自动将其他记录的 `is_default` 改为0

### 销售终端
1）字段说明：`name`（终端名称）、`code`（终端编号，唯一）、`is_default`（默认终端：1=是，0=否）、`db_no`（终端独立数据库编号，可为空）
2）默认数据：预写入一条 `name=门店终端001`，`code=T001`，`is_default=1`
3）业务说明：按名称/编号关键词查询、新增、修改、删除
4）无状态字段
5）`is_default` 设为1时，自动将其他记录的 `is_default` 改为0

# 实现

## step 1: 在 `pg` 目录下，新建 `10_create_staff_terminal_tables.sql`
3张表全部写在同一个文件中：

- **`biz_purchase_staff`**（采购业务员）：`id SERIAL PK`, `name VARCHAR(50) NOT NULL`, `phone VARCHAR(50) NOT NULL DEFAULT ''`, `address VARCHAR(200) NOT NULL DEFAULT ''`, `is_default SMALLINT NOT NULL DEFAULT 0`, `purchase_limit NUMERIC(12,2) NOT NULL DEFAULT 0`, `created_at TIMESTAMP DEFAULT NOW()`；预写入1条默认数据
- **`biz_sales_staff`**（销售业务员）：`id SERIAL PK`, `name VARCHAR(50) NOT NULL`, `phone VARCHAR(50) NOT NULL DEFAULT ''`, `address VARCHAR(200) NOT NULL DEFAULT ''`, `is_default SMALLINT NOT NULL DEFAULT 0`, `commission_rate NUMERIC(5,4) NOT NULL DEFAULT 0`, `created_at TIMESTAMP DEFAULT NOW()`；预写入1条默认数据
- **`biz_terminal`**（销售终端）：`id SERIAL PK`, `name VARCHAR(100) NOT NULL`, `code VARCHAR(50) NOT NULL UNIQUE`, `is_default SMALLINT NOT NULL DEFAULT 0`, `db_no VARCHAR(100) NOT NULL DEFAULT ''`, `created_at TIMESTAMP DEFAULT NOW()`；预写入1条默认数据

同步更新 `00_reinit_db.sh`，追加执行 `10_create_staff_terminal_tables.sql` 的 step 10。

## step 2: 后端 `jxcServer` 工程，新建 `api/staff.py`
3种资源的接口全部写在同一个文件 `api/staff.py` 中，Blueprint 名 `staff_bp`：

**采购业务员** `/api/purchase-staff/`：
- `GET /api/purchase-staff/list`：支持 `keyword` 关键词（同时匹配 name、phone）
- `POST /api/purchase-staff/add`：Body `{name, phone, address, is_default, purchase_limit}`，name 不能为空；is_default=1 时清空其他记录的 is_default
- `PUT /api/purchase-staff/update`：Body `{id, name, phone, address, is_default, purchase_limit}`；is_default=1 时清空其他记录的 is_default
- `DELETE /api/purchase-staff/delete`：Body `{id}`

**销售业务员** `/api/sales-staff/`：
- `GET /api/sales-staff/list`：支持 `keyword` 关键词（同时匹配 name、phone）
- `POST /api/sales-staff/add`：Body `{name, phone, address, is_default, commission_rate}`，name 不能为空；is_default=1 时清空其他记录的 is_default
- `PUT /api/sales-staff/update`：Body `{id, name, phone, address, is_default, commission_rate}`；is_default=1 时清空其他记录的 is_default
- `DELETE /api/sales-staff/delete`：Body `{id}`

**销售终端** `/api/terminal/`：
- `GET /api/terminal/list`：支持 `keyword` 关键词（同时匹配 name、code）
- `POST /api/terminal/add`：Body `{name, code, is_default, db_no}`，name/code 不能为空；is_default=1 时清空其他记录的 is_default
- `PUT /api/terminal/update`：Body `{id, name, code, is_default, db_no}`；is_default=1 时清空其他记录的 is_default
- `DELETE /api/terminal/delete`：Body `{id}`

在 `app.py` 中注册：`from api.staff import staff_bp` + `app.register_blueprint(staff_bp, url_prefix='/api')`

完成后用 curl 对3种资源各做一次完整的增删改查验证。

## step 3: 前端 `jxcClient` 实现3个页面
3个页面风格参考 `AreaManage.vue`（增删改查列表页），无状态列。

- 新建 `src/views/PurchaseStaffManage.vue`（采购业务员管理）
  - 列：序号、姓名、电话、地址、单次采购金额上限、默认采购员（tag标识）、操作
  - 搜索：姓名或电话关键词（`keyword`）
  - 表单：姓名（必填）、电话、地址、单次采购金额上限（number input，0表示不限）、默认采购员（radio：是/否）
- 新建 `src/views/SalesStaffManage.vue`（销售业务员管理）
  - 列：序号、姓名、电话、地址、销售提成比例、默认销售员（tag标识）、操作
  - 搜索：姓名或电话关键词（`keyword`）
  - 表单：姓名（必填）、电话、地址、销售提成比例（number input，如输入5代表5%，存储0.05）、默认销售员（radio：是/否）
- 新建 `src/views/TerminalManage.vue`（销售终端管理）
  - 列：序号、终端名称、终端编号、数据库编号、默认终端（tag标识）、操作
  - 搜索：名称或编号关键词（`keyword`）
  - 表单：终端名称（必填）、终端编号（必填）、数据库编号、默认终端（radio：是/否）

在 `Home.vue` 的 `COMPONENT_MAP` 中注册3个映射：
```js
'/system/basic/purchase-staff': PurchaseStaffManage,
'/system/basic/sales-staff':    SalesStaffManage,
'/system/basic/branch':         TerminalManage,
```

对应 api 文件：
- `src/api/purchaseStaff.js`
- `src/api/salesStaff.js`
- `src/api/terminal.js`
