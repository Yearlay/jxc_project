# 商品列表

**功能**：实现三级菜单页面
- 商品管理 → 商品列表

---
# 输入
1）UI页面风格参考 `CustomerManage.vue`（增删改查列表页），无状态列。
2）[商品]字段说明：`序号`、`编码`、`其他编码`、`名称`、`分类`、`采购价`、`销售价`、`会员价`、`批发价`、`单位`、`生产厂家`、`当前库存`、`库存下限`、`保质期`、`参与积分`(是/否)、`会员折扣`(是/否)。
3）[商品]业务说明：按名称/编号查询、按分类查询、新建商品、编辑、删除。
4）[商品库存]字段说明：`存放仓库`、`规格`、`批号`、`尺码`、`颜色`、`生产日期`、`有效日期`、`商品名称`、`当前库存`
5）[商品库存]业务说明：增加库存、库存盘点、仓库调拨。

# 实现
## step 1: 在 `pg` 目录下，更新【商品列表】相关的表结构
1）参考[商品]字段说明，设计[商品列表]数据库中的表结构。补充说明：
- 分类：关联 goods_category.id
- 采购价: 集成本价，float类型，价格保留小数点后两位。
- 销售价：卖给普通顾客（非任何会员）的销售价。
- 会员价：当识别到顾客为会员，采用 会员价*折扣率 售卖。
- 批发价：对外批发的价格，当 POS前台销售，标识为批发时，采用批发价进行售卖。
- 单位：关联 biz_unit.id
- 生产厂家：关联 biz_manufacturer.id
- 当前库存：各个仓库的库存的数量之和，新建和编辑对话框中不可编辑；若库存>0，不可删除。
- 库存下限：库存不足的预警值，当`当前库存`小于或等于`库存下限`，对应的商品一栏显示为红色字体。
- 保质期：数字，单位为天；
- 参与积分: 1-参与积分；0-不参与积分；UI端显示“是/否”
- 会员折扣: 1-参与会员折扣；0-不参与会员折扣；UI端显示“是/否”
备注：不设计 状态列（启用/禁用）

2）[商品库存]字段说明补充如下：
- 存放仓库：关联 biz_warehouse.id
- 规格+批号+尺码+颜色: 文字输入
- 生产日期：日期格式
- 有效日期：按 `生产日期`+商品`保质期`，计算出有效日期。
- 商品名称：关联 商品表
- 当前库存：int整数。

## step 2: 后端 `jxcServer` 工程，创建【商品列表】相关 API
参考[商品]业务说明，设计[商品]相关业务的接口。
参考[商品库存]业务说明，设计[商品库存]相关业务的接口。

## step 3: 前端 `jxcClient` 实现 "商品列表" 菜单页面
UI页面风格参考 `AreaManage.vue`（增删改查列表页），无状态列。
- 上部分UI：[按名称/编号查询]搜索框。新建商品按钮（配加号的icon）
- 左侧分类列表：采用 a-tree 展示商品分类，当点击单个分类，按分类查询商品列表。
- 右侧列表：商品列表，至少展示列：序号、编码、名称、类型、采购价、销售价、会员价、批发价、当前库存、单位、保质期、生产厂家；左右侧显示三点图标操作菜单：编辑、删除（库存为0显示；库存>0不显示）、库存清单、增加库存、库存盘点(库存>0显示；库存=0不显示)、仓库调拨。点击菜单都弹出对话框。
- `新建/编辑`对话框：根据[商品]字段说明，AI自行设计。
- `删除`对话框：AI自行设计。
- `库存清单`对话框：商品列表，字段全部展示。
- `增加库存`对话框: 根据[商品库存]字段说明，AI自行设计。
- `库存盘点`对话框: 先选择有库存的仓库，然后对[商品库存]的各个字段进行编辑。
- `仓库调拨`对话框: 先选择[源仓库]，在选择[目标仓库]，显示[源仓库]库存数量，输入调拨数量(需小于[源仓库]库存数量)；点击[调拨]按钮，完成调拨。点击[取消]按钮，取消并关闭对话框。
