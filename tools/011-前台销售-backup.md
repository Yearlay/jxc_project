# 前台销售

**功能**：实现菜单页面 PosPage.vue
- POS 前台销售界面：零售商品销售界面

---
# 输入
1）[前台销售]字段说明：`销售单号`、`仓库`、`业务员`、`操作员`、`终端`、`顾客信息`、`商品列表`、`商品总数`、`总金额`、`折后价`、`付款方式`、`支付金额`、`应找金额`、`状态`、`日期`
2）[前台销售]业务说明：挂单，取单，结算。

# 实现
## step 1: 在 `pg` 目录下，设计【前台销售】相关数据表。
1）【仓库】表添加“默认仓库”字段（是/否）；并将“北京总仓”设置为默认仓库。
2）参考[前台销售]字段说明，设计[前台销售]表结构。补充说明：
- 销售单号：`P-YYMMDD`+当前时间的毫秒数值。
- 仓库：必填，缺省为默认仓库。选择仓库为对话框，效果AI自行设计。
- 业务员：必填，关联的是销售业务员，选择【销售业务员】中的默认销售员。
- 操作员：必填，为当前登录的用户。
- 终端：必填，关联的是销售终端，选择【销售终端】中的默认终端。
- 顾客信息：关联客户信息表biz_customer。
- 商品列表：存储 `商品id` + `数量`的商品列表。
- 商品总数：商品列表中的数量之和。
- 总金额：按商品的`销售价`和商品数量，计算出总金额。
- 折后价：按照客户的`会员类型`的折扣率，以及商品`会员折扣`，综合判断和计算。
- 付款方式：默认“现金”支付
- 支付金额：输入金额数字
- 应找金额：自动计算 `支付金额 - 折后价`
- 状态：待结算、已结算
- 日期：创建或更新日期

## step 2: 后端 `jxcServer` 工程，创建【前台销售】相关 API
参考[前台销售]业务说明，设计[前台销售]相关业务的接口。

## step 3: 前端 `jxcClient` 实现 "前台销售" 菜单页面
要求：一页展示；不要上下滚动; 字体设计为小号。
分上、中、下三个区域：
- 上区域（布局的时候考虑高度，尽可能压缩高度）：水平五五分布局；左侧展示`销售单号`、`操作仓库`、`业务员`、`操作员`、`终端`；右侧展示：`顾客信息`（名称、余额、积分、会员号、会员类型、折扣率）
- 中区域：水平八二分布局
中左侧展示：【商品列表【序号、商品名称、数量、单位、原价、折扣、折后价、仓库、参与积分、参与折扣（上下健支持选择）
中右侧展示：【操作快捷键】`+ 结算`、`F8 挂单`、`F9 取单`、`F12-全删`、`UP/DOWN 选择商品`、`Delete 删除`、`F1 改数量`、`F4 改顾客`
- 下区域（布局的时候考虑高度，尽可能压缩高度）：水平 3-5-2布局
下左侧展示：鼠标默认选中；输入商品名称或条形码；按回车在下中侧展示查找到的商品内容（鼠标移动到下中侧区域）；
下中侧展示：查询到的商品列表（编号，名称，价格）；上下按键和选择，回车按键添加商品到 中左侧商品列表；清空查询列表；鼠标移动到 下左侧输入框中。
下右侧展示：总金额、总数、折后价格。

弹框设计：
1）AI自行设计：仓库选择弹框、业务员选择弹框、客户选择弹框、挂单确认弹框、取单选择弹框、改数量弹框、F12全删确认弹框。
2）结算弹框：需显示`总金额`、`折后金额`、`付款方式`、`支付金额`、`应找金额`（自动计算）、`备注`（文字输入）、取消按钮、确认结算按钮。
