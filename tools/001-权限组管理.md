# 权限组管理 相关业务

**功能**：实现权限组管理菜单页；权限组相关数据库表设计；权限组相关的服务端接口；前端权限组管理页面。

---

# 输入

## 权限组说明

1. 系统默认两个权限组：【管理员】和【普通成员】。
2. `admin` 用户默认分配到【管理员】权限组。
3. 每个权限组可配置可访问的菜单列表，以及包含的成员。

## 数据库表设计

### sys_group（权限组表）
| 字段         | 类型          | 说明                  |
|------------|-------------|---------------------|
| id         | SERIAL PK   | 主键                  |
| name       | VARCHAR(50) | 权限组名称，唯一            |
| remark     | VARCHAR(200)| 备注（可为空）             |
| status     | SMALLINT    | 状态：1=启用 0=禁用        |
| created_at | TIMESTAMP   | 创建时间                |

### sys_group_menu（权限组菜单关联表）
| 字段       | 类型        | 说明             |
|----------|-----------|----------------|
| id       | SERIAL PK | 主键             |
| group_id | INT       | 关联 sys_group.id |
| menu_id  | INT       | 关联 sys_menu.id  |
| UNIQUE   | (group_id, menu_id) | 联合唯一     |

### sys_group_user（权限组成员关联表）
| 字段       | 类型        | 说明              |
|----------|-----------|-----------------|
| id       | SERIAL PK | 主键              |
| group_id | INT       | 关联 sys_group.id  |
| user_id  | INT       | 关联 sys_user.id   |
| UNIQUE   | (group_id, user_id) | 联合唯一      |

> 注意：同时需修改 `sys_user` 表，增加 `group_id INT DEFAULT NULL` 字段，记录用户所属权限组（主权限组）。

## 后端 API 接口设计

基础路径：`/api/group`

| 方法     | 路径                       | 说明              |
|--------|--------------------------|-----------------|
| GET    | /api/group/list          | 权限组列表（含成员数量）    |
| POST   | /api/group/add           | 新增权限组           |
| PUT    | /api/group/rename        | 重命名权限组          |
| DELETE | /api/group/delete        | 删除权限组           |
| GET    | /api/group/menus         | 权限组已配置的菜单 ID 列表 |
| POST   | /api/group/menus/save    | 保存权限组菜单配置       |
| GET    | /api/group/members       | 权限组成员列表         |
| POST   | /api/group/members/add   | 权限组添加成员         |
| DELETE | /api/group/members/remove| 权限组删除成员         |

请求参数说明：
- 列表/详情类接口：使用 Query 参数传 `group_id`。
- 新增/编辑类接口：使用 JSON Body。
- 删除类接口：使用 JSON Body 传 `id` 或 `group_id + user_id`。

## 权限组管理页 UI 说明

整体布局：左侧为权限组列表区，右侧为 Tab 区（权限配置 + 成员管理），效果参考 `design/效果图/权限组管理UI效果.png`。

### 左侧：权限组列表区
- 列表可滚动，每个 Item 显示权限组名称和成员数量。
- 选中 Item 高亮，右侧联动显示对应权限组详情。
- 每个 Item 右上角显示三点图标（`EllipsisOutlined`），使用 `a-dropdown` + `a-menu` 实现下拉菜单：
  - **重命名**：弹出内联输入框或 `a-modal` 编辑名称。
  - **删除**：`a-popconfirm` 二次确认后删除（不允许删除有成员的权限组）。
- 列表底部常驻 **「+ 添加权限组」** 按钮，点击弹出 `a-modal` 输入名称。

### 右侧 Tab：权限配置
- 使用 `a-tabs` 切换。
- 展示全部菜单树（三级），每一项配有 `a-checkbox`。
- 顶部有「全选」复选框，支持全选/全不选。
- 底部有「保存配置」按钮，调用 `/api/group/menus/save`。

### 右侧 Tab：成员管理
- 顶部显示 **「+ 添加成员」** 按钮，点击弹出 `a-modal` 展示可添加的用户列表（排除已在组的成员）。
- 成员列表展示：头像（`a-avatar`）+ 姓名 + 用户名。
- 选中某个成员后，显示 **「移除成员」** 按钮（红色），`a-popconfirm` 二次确认后删除。

# 实现

## step 1: 在 `pg` 目录下，创建权限组相关的表结构

1. 文件命名：按已有序号顺序创建，如 `04_create_group_tables.sql`。
2. 在该文件中创建 `sys_group`、`sys_group_menu`、`sys_group_user` 三张表，并添加字段注释。
3. 初始化数据：
   - 插入权限组【管理员】和【普通成员】。
   - 【管理员】权限组关联全部 85 条菜单记录（从 `sys_menu` 中读取插入）。
   - `admin` 用户关联到【管理员】权限组。
4. 执行命令（参考已有文件头部注释格式）：
   ```
   psql -U onepiece -d jxc -h 192.168.0.116 -p 5432 -W -f 04_create_group_tables.sql
   ```

## step 2: 后端 `jxcServer` 工程，创建权限组相关 API

1. 新建文件 `server/jxcServer/api/group.py`，注册蓝图 `group_bp`，前缀 `/api/group`。
2. 在 `app.py` 中注册 `group_bp`。
3. 按接口设计实现全部 9 个接口，统一返回格式 `{"code": 200, "msg": "...", "data": ...}`。
4. 删除权限组时，若该组下存在成员，返回错误提示，拒绝删除。
5. 接口完成后，使用 `curl` 逐一验证。

## step 3: 前端 `jxcClient` 安装图标包并为菜单配置 icon

1. 安装：`npm install @ant-design/icons-vue`。
2. 在 `SideMenu.vue` 中根据菜单名称（或 `path`）映射对应的图标组件。
3. 菜单图标参考：
   - 基础档案 → `DatabaseOutlined`
   - 采购管理 → `ShoppingCartOutlined`
   - 销售管理 → `ShopOutlined`
   - 库存管理 → `InboxOutlined`
   - 报表查询 → `BarChartOutlined`
   - 系统维护 → `SettingOutlined`
   - 权限管理 → `SafetyOutlined`

## step 4: 前端 `jxcClient` 实现权限组管理页面

1. 新建页面 `src/views/GroupManage.vue`，在 `router/index.js` 中注册路由，路径与菜单中"权限组管理"对应的 `path` 一致。
2. 布局：左右两栏，左侧固定宽度（如 240px），右侧自适应填充。
3. 左侧权限组列表：
   - 组件提取为 `src/components/GroupList.vue`。
   - 使用 `a-dropdown` + `EllipsisOutlined` 实现重命名/删除操作。
   - 底部固定「+ 添加权限组」按钮。
4. 右侧 Tab 区：
   - 权限配置 Tab：菜单树 + checkbox，含「全选」和「保存配置」。
   - 成员管理 Tab：成员列表 + 添加/移除成员操作。
5. 所有操作完成后刷新对应区域数据，使用 `a-message` 提示操作结果。
6. 图标均使用 `@ant-design/icons-vue` 中的组件。
