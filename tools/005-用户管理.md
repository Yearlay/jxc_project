# 输入

## UI 风格参考
效果图参考：`design/效果图/基础信息设置_片区管理.png`（用户管理与片区管理页面风格接近）

- 顶部：搜索框（请输入用户名称）+ 「查询」+ 「创建新用户」按钮（`PlusOutlined` 图标）

## 字段说明（sys_user 表扩展）

**现有字段**（`pg/02_create_tables.sql`）：

| 字段       | 类型         | 说明                             |
|------------|--------------|----------------------------------|
| id         | SERIAL PK    | 主键                             |
| username   | VARCHAR(50)  | 登录用户名（唯一）               |
| password   | VARCHAR(64)  | 登录密码（MD5）                  |
| real_name  | VARCHAR(50)  | 真实姓名                         |
| role       | VARCHAR(20)  | 角色：admin=管理员 staff=普通用户 |
| status     | SMALLINT     | 状态：1=启用 0=禁用              |
| created_at | TIMESTAMP    | 创建时间                         |

**新增字段**：

| 字段         | 类型    | 说明                                    |
|--------------|---------|------------------------------------------|
| warehouse_id | INT     | 操作仓库（FK → biz_warehouse.id，可为空） |
| area_id      | INT     | 管辖片区（FK → biz_area.id，可为空）     |

## 业务说明

- 列表支持按用户名关键词搜索（ILIKE 任意匹配），全量展示不分页。
- 列表展示字段：用户名、姓名、角色（Tag：admin=蓝色/staff=默认）、操作仓库名称、管辖片区名称、状态（Tag：启用=绿/禁用=灰）。
- **创建新用户**：弹窗填写用户名（必填，唯一）、姓名、初始密码（必填）、角色（下拉）、操作仓库（下拉，可不选）、管辖片区（下拉，可不选）、状态。
- **修改用户**：与创建使用同一弹窗，回填数据（不含密码字段）。
- **密码重置**：独立弹窗，输入新密码 + 确认密码，提交后后端 MD5 存储。
- **删除**：`Modal.confirm()` 二次确认，不能删除当前登录用户自己；
- 密码在后端 MD5 加密存储，前端传明文。
- 新建/修改时 `username` 唯一，重复时返回友好提示。

## 接口设计

基础路径：`/api/user`

| 方法   | 路径                    | 说明                                     |
|--------|-------------------------|------------------------------------------|
| GET    | /api/user/manage/list   | 用户列表（支持 username 关键词搜索）     |
| POST   | /api/user/manage/add    | 新建用户                                 |
| PUT    | /api/user/manage/update | 修改用户信息（不含密码）                 |
| PUT    | /api/user/manage/reset-password | 重置用户密码                   |
| DELETE | /api/user/manage/delete | 删除用户                                 |

> 原有 `GET /api/user/list`（权限组用）保持不变，新接口在 `/api/user/manage/` 前缀下。

# 实现

## step 1: 更新 `pg/02_create_tables.sql`，并执行 `00_reinit_db.sh` 重建数据库

1. 在 `pg/02_create_tables.sql` 的 `sys_user` 建表语句末尾追加两个字段：
   ```sql
   warehouse_id  INT  REFERENCES biz_warehouse(id) ON DELETE SET NULL,
   area_id       INT  REFERENCES biz_area(id) ON DELETE SET NULL,
   ```
   并添加对应字段注释（`COMMENT ON COLUMN`）。

2. 因为 `biz_warehouse`/`biz_area` 在 step5/step7 才建表，而 `sys_user` 在 step2，
   **外键约束不能在建表时直接添加**。解决方案：在 `02_create_tables.sql` 中先不加外键，
   改为在 `07_create_warehouse_table.sql`（最后执行的脚本）末尾追加 ALTER TABLE：
   ```sql
   ALTER TABLE sys_user
     ADD COLUMN IF NOT EXISTS warehouse_id INT,
     ADD COLUMN IF NOT EXISTS area_id      INT;
   ALTER TABLE sys_user
     ADD CONSTRAINT fk_user_warehouse FOREIGN KEY (warehouse_id)
       REFERENCES biz_warehouse(id) ON DELETE SET NULL;
   ALTER TABLE sys_user
     ADD CONSTRAINT fk_user_area FOREIGN KEY (area_id)
       REFERENCES biz_area(id) ON DELETE SET NULL;
   COMMENT ON COLUMN sys_user.warehouse_id IS '操作仓库ID（关联biz_warehouse）';
   COMMENT ON COLUMN sys_user.area_id      IS '管辖片区ID（关联biz_area）';
   ```

3. 执行 `00_reinit_db.sh` 重建数据库并验证 `sys_user` 表结构包含新字段：
   ```
   cd pg && bash 00_reinit_db.sh
   PGPASSWORD=yearlay_1987 psql -U onepiece -d jxc -h 192.168.0.116 -p 5432 \
     -c "\d sys_user"
   ```

## step 2: 后端 `jxcServer` 工程，扩展用户管理 API

1. 修改 `server/jxcServer/api/user.py`，在现有 `GET /api/user/list` 基础上新增 5 个接口：
   - `manage/list`：LEFT JOIN biz_warehouse / biz_area 返回 warehouse_name / area_name；支持 `username` 关键词搜索（ILIKE）。
   - `manage/add`：username/password/real_name 必填；password MD5 加密；username 唯一校验；支持 warehouse_id/area_id 可空。
   - `manage/update`：id/username/real_name 必填；username 唯一校验（排除自身）；不更新密码。
   - `manage/reset-password`：id/password 必填；MD5 加密后更新。
   - `manage/delete`：id 必填；不允许删除 id=1（超级管理员保护，或从 token 取当前用户 id 禁止自删）。
2. 密码 MD5 复用 `auth.py` 中的 `_md5` 函数（从 auth 导入或直接在 user.py 中定义同名函数）。
3. 接口完成后使用 `curl` 验证全部 5 个新接口。

## step 3: 前端 `jxcClient` 实现「用户管理」页面

文件：`src/views/UserManage.vue`（**覆盖**现有文件，原文件内容仅为占位）

同时需要在 `src/api/user.js`（**新建或追加**）中封装 5 个新接口。

### 整体布局（参考片区管理页面风格）
```
┌────────────────────────────────────────────────────────────┐
│ [搜索框 请输入用户名称] [查询]            [创建新用户 +]     │
├────────────────────────────────────────────────────────────┤
│ 用户名 │ 姓名 │ 角色 │ 操作仓库 │ 管辖片区 │ 状态 │ 操作   │
│ admin │ 管理员│admin│ 北京总仓 │ 北京片区 │ 启用 │  ···   │
└────────────────────────────────────────────────────────────┘
```

### 实现要点

1. **顶部工具栏**：搜索框（`@pressEnter` + `@change` 清空时自动刷新），「创建新用户」按钮（`PlusOutlined`）。
2. **列表 `a-table`**：列为用户名、姓名、角色（`a-tag`，admin=蓝/staff=默认）、操作仓库、管辖片区、状态（`a-tag`，启用=绿/禁用=灰），不分页。
3. **操作列** `EllipsisOutlined` 下拉菜单：
   - 「修改」`EditOutlined`
   - 「密码重置」`KeyOutlined`
   - 「删除」`DeleteOutlined`，红色，`Modal.confirm()`
4. **新建/修改弹窗**：
   - 字段：用户名（必填）、姓名、密码（新建必填/修改不显示）、角色（`a-select`：管理员/普通用户）、操作仓库（`a-select`，可清空，数据来自 `getWarehouseList()`）、管辖片区（`a-select`，可清空，数据来自 `getAreaList()`）、状态（`a-radio-group`）
   - 打开弹窗时同步加载仓库/片区下拉数据
5. **密码重置弹窗**：新密码（必填，≥6位）+ 确认密码（必填，与新密码一致）
6. 在 `Home.vue` 的 `COMPONENT_MAP` 中注册：`'/system/user/manage': UserManage`

# BUG和优化点：
1）UI优化：admin用户是系统默认用户，不显示三点图标，不可操作。
2）业务优化：新建和编辑用户界面，操作仓库和管辖片区，支持多选。
3）交互优化：管辖片区中”未划分“为缺失值，新建和编辑界面不显示”未划分“。
4）UI优化：删除【退出登录】左边的本文；并将【退出登录】替换为 当前登录的用户的用户名；用户名右边加icon <down-outlined />，鼠标放上去，向下冒泡弹出 退出登录，点击后退出登录。
