# 计量+厂家+车辆+付款

**功能**：实现三级菜单页面（parent_id=60，基础信息设置）
- 系统维护 → 基础信息设置 → 计量单位（菜单 id=62，path=/system/basic/unit）
- 系统维护 → 基础信息设置 → 厂家信息（菜单 id=63，path=/system/basic/manufacturer）
- 系统维护 → 基础信息设置 → 车辆信息（菜单 id=66，path=/system/basic/vehicle）
- 系统维护 → 基础信息设置 → 付款方式（菜单 id=67，path=/system/basic/payment）

---
# 输入

### 计量单位
1）字段说明：`name`（计量单位名称，唯一）
2）默认数据：个、件、瓶、包、箱、KG、升、套
3）业务说明：按名称关键词查询、新增、修改、删除（删除前检查是否被商品引用，已引用禁止删除）
4）无状态字段

### 厂家信息
1）字段说明：`name`（厂家名称，唯一）、`phone`（电话）、`address`（地址）、`license`（营业执照）、`bank_account`（银行账户）
2）默认数据：不初始化
3）业务说明：按名称/电话关键词查询、新增、修改、删除
4）无状态字段

### 车辆信息
1）字段说明：`plate`（车牌号，唯一）、`owner`（车主）、`owner_phone`（车主电话）、`remark`（备注）
2）默认数据：不初始化
3）业务说明：按车牌号/车主/车主电话关键词查询、新增、修改、删除
4）无状态字段

### 付款方式
1）字段说明：`name`（付款方式名称，唯一）
2）默认数据：现金、信用卡、支付宝、微信、银行转账
3）业务说明：按名称关键词查询、新增、修改、删除（删除前检查是否被单据引用，已引用禁止删除）
4）无状态字段

# 实现

## step 1: 在 `pg` 目录下，新建 `09_create_basic_tables.sql`
4张表全部写在同一个文件中：

- **`biz_unit`**（计量单位）：`id SERIAL PK`, `name VARCHAR(20) NOT NULL UNIQUE`, `created_at TIMESTAMP DEFAULT NOW()`；初始化8条默认数据
- **`biz_manufacturer`**（厂家信息）：`id SERIAL PK`, `name VARCHAR(100) NOT NULL UNIQUE`, `phone VARCHAR(50) DEFAULT ''`, `address VARCHAR(200) DEFAULT ''`, `license VARCHAR(100) DEFAULT ''`, `bank_account VARCHAR(100) DEFAULT ''`, `created_at TIMESTAMP DEFAULT NOW()`；不初始化数据
- **`biz_vehicle`**（车辆信息）：`id SERIAL PK`, `plate VARCHAR(20) NOT NULL UNIQUE`, `owner VARCHAR(50) DEFAULT ''`, `owner_phone VARCHAR(50) DEFAULT ''`, `remark VARCHAR(200) DEFAULT ''`, `created_at TIMESTAMP DEFAULT NOW()`；不初始化数据
- **`biz_payment`**（付款方式）：`id SERIAL PK`, `name VARCHAR(50) NOT NULL UNIQUE`, `created_at TIMESTAMP DEFAULT NOW()`；初始化5条默认数据

同步更新 `00_reinit_db.sh`，追加执行 `09_create_basic_tables.sql` 的 step 9。

## step 2: 后端 `jxcServer` 工程，新建 `api/basic.py`
4种资源的接口全部写在同一个文件 `api/basic.py` 中，Blueprint 名 `basic_bp`：

**计量单位** `/api/unit/`：
- `GET /api/unit/list`：支持 `name` 关键词查询
- `POST /api/unit/add`：Body `{name}`，名称不能为空
- `PUT /api/unit/update`：Body `{id, name}`
- `DELETE /api/unit/delete`：Body `{id}`

**厂家信息** `/api/manufacturer/`：
- `GET /api/manufacturer/list`：支持 `keyword` 关键词（同时匹配 name、phone）
- `POST /api/manufacturer/add`：Body `{name, phone, address, license, bank_account}`，name 不能为空
- `PUT /api/manufacturer/update`：Body `{id, name, phone, address, license, bank_account}`
- `DELETE /api/manufacturer/delete`：Body `{id}`

**车辆信息** `/api/vehicle/`：
- `GET /api/vehicle/list`：支持 `keyword` 关键词（同时匹配 plate、owner、owner_phone）
- `POST /api/vehicle/add`：Body `{plate, owner, owner_phone, remark}`，plate 不能为空
- `PUT /api/vehicle/update`：Body `{id, plate, owner, owner_phone, remark}`
- `DELETE /api/vehicle/delete`：Body `{id}`

**付款方式** `/api/payment/`：
- `GET /api/payment/list`：支持 `name` 关键词查询
- `POST /api/payment/add`：Body `{name}`，名称不能为空
- `PUT /api/payment/update`：Body `{id, name}`
- `DELETE /api/payment/delete`：Body `{id}`

在 `app.py` 中注册：`from api.basic import basic_bp` + `app.register_blueprint(basic_bp, url_prefix='/api')`

完成后用 curl 对4种资源各做一次完整的增删改查验证。

## step 3: 前端 `jxcClient` 实现4个页面
4个页面风格参考 `AreaManage.vue`（增删改查列表页），无状态列。

- 新建 `src/views/UnitManage.vue`（计量单位管理）
  - 列：序号、名称、操作（修改/删除）
  - 搜索：名称关键词
- 新建 `src/views/ManufacturerManage.vue`（厂家信息管理）
  - 列：序号、厂家名称、电话、地址、营业执照、银行账户、操作
  - 搜索：名称或电话关键词（`keyword`）
- 新建 `src/views/VehicleManage.vue`（车辆信息管理）
  - 列：序号、车牌号、车主、车主电话、备注、操作
  - 搜索：车牌号/车主/车主电话关键词（`keyword`）
- 新建 `src/views/PaymentManage.vue`（付款方式管理）
  - 列：序号、名称、操作（修改/删除）
  - 搜索：名称关键词

在 `Home.vue` 的 `COMPONENT_MAP` 中注册4个映射：
```js
'/system/basic/unit':         UnitManage,
'/system/basic/manufacturer': ManufacturerManage,
'/system/basic/vehicle':      VehicleManage,
'/system/basic/payment':      PaymentManage,
```
