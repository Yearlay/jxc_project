# 客户管理

**功能**：实现二级菜单页面
- 系统维护 → 客户管理（菜单路径：`/system/customer`）

---

# 输入

## UI 风格参考
效果图：`design/效果图/客户管理.png` 和 `design/效果图/客户管理_创建新客户.png`

- 顶部：「客户信息」标签 + 搜索框（请输入客户名称/联系人/电话）+ 「查询」按钮 + 「创建新客户」按钮
- 左侧：「片区管理」树形列表（所有区域 → 各片区），点击片区过滤右侧列表
- 右侧：客户列表表格 + 分页
- 操作：每行末尾三点图标，下拉菜单含「修改」「充值」「删除」

## 字段说明

| 字段           | 类型            | 说明                                       |
|----------------|-----------------|--------------------------------------------|
| id             | SERIAL PK       | 主键                                       |
| code           | VARCHAR(50)     | 客户编号，唯一（如 C2024001）              |
| name           | VARCHAR(100)    | 客户名称（必填）                           |
| contact        | VARCHAR(50)     | 联系人                                     |
| phone          | VARCHAR(20)     | 电话号码（必填，唯一）                     |
| address        | VARCHAR(200)    | 地址                                       |
| area_id        | INTEGER FK      | 所属片区（关联 `biz_area.id`）             |
| salesman       | INTEGER FK      | 所属业务员（关联 `sys_user.id`）                     |
| member_type_id | INTEGER FK      | 会员类型（关联 `biz_member_type.id`）      |
| member_no      | VARCHAR(50)     | 会员号（必填，唯一，如 M2024001）          |
| birthday       | DATE            | 会员生日（可为空）                         |
| points         | INTEGER         | 积分（默认 0）                             |
| balance        | NUMERIC(10,2)   | 账户余额（默认 0.00）                      |
| status         | SMALLINT        | 状态：1=启用 0=禁用                        |
| created_at     | TIMESTAMP       | 创建时间                                   |

## 业务说明

- 列表支持关键词搜索（客户名称 / 联系人 / 电话 / 会员号，ILIKE 任意匹配）。
- 支持按片区筛选：点击左侧片区树节点，右侧列表过滤；点击「所有区域」还原全部。
- 支持分页：默认 10 条/页。
- 创建新客户：弹窗填写基本信息 + 会员信息。
- 修改客户：与创建使用同一弹窗，回填已有数据。
- 充值：独立弹窗，仅填写充值金额（正数），后端累加到 `balance`。
- 删除：`Modal.confirm()` 二次确认（避免 popconfirm+dropdown 嵌套 BUG）。
- 会员类型在列表中以彩色标签展示（金卡=gold，银卡=blue，普通=default，其他=green）。
- `code`、`member_no` 若前端不填，后端自动生成（格式 `C{年}{序号4位}` / `M{年}{序号4位}`）。

## 接口设计

基础路径：`/api/customer`

| 方法   | 路径                      | 说明                                              |
|--------|---------------------------|---------------------------------------------------|
| GET    | /api/customer/list        | 客户列表（支持 keyword、area_id、page、page_size）|
| POST   | /api/customer/add         | 新建客户                                          |
| PUT    | /api/customer/update      | 修改客户信息（含所有字段）                        |
| PUT    | /api/customer/recharge    | 充值（仅传 id + amount，后端累加 balance）        |
| DELETE | /api/customer/delete      | 删除客户                                          |

# 实现

## step 1: 在 `pg` 目录下，创建客户管理表结构并初始化数据

文件命名：`06_create_customer_table.sql`

1. 创建 `biz_customer` 表，字段及约束参考【字段说明】，添加字段注释。
2. 建立外键：`area_id → biz_area(id)`，`member_type_id → biz_member_type(id)`，均设为 `ON DELETE SET NULL`。
3. 初始化 5~8 条示例数据（覆盖不同片区 / 会员类型），参考效果图数据风格。
4. 将新文件加入 `00_reinit_db.sh` 的执行序列（step6）。
5. 执行验证：
   ```
   psql -U onepiece -d jxc -h 192.168.0.116 -p 5432 -W -f 06_create_customer_table.sql
   ```

## step 2: 后端 `jxcServer` 工程，创建客户管理 API

1. 新建 `server/jxcServer/api/customer.py`，注册蓝图 `customer_bp`，前缀 `/api/customer`，实现 5 个接口。
2. `list` 接口：
   - 支持 `keyword`（ILIKE 匹配 name / contact / phone / member_no）
   - 支持 `area_id`（精确匹配，为空或为 0 时不过滤）
   - 支持 `page`（默认 1）、`page_size`（默认 10）分页
   - LEFT JOIN `biz_area` 和 `biz_member_type`，返回 `area_name`、`member_type_name`
   - 返回 `{ list: [...], total: N }`
3. `add` / `update` 接口：`code` 和 `member_no` 为空时后端自动生成
4. `recharge` 接口：接收 `{ id, amount }`，校验 amount > 0，执行 `balance = balance + amount`
5. `delete` 接口：直接删除，无关联检查（预留）
6. 在 `app.py` 中导入并注册 `customer_bp`
7. 统一返回格式 `{"code": 200, "msg": "...", "data": ...}`
8. 接口完成后使用 `curl` 验证全部 5 个接口

## step 3: 前端 `jxcClient` 实现「客户管理」页面

文件：`src/views/CustomerManage.vue`，新建 API 文件 `src/api/customer.js`

### 整体布局（参考效果图）
```
┌──────────────────────────────────────────────────────────┐
│ 客户信息  [搜索框 请输入客户名称/联系人/电话]  [查询]  [创建新客户] │
├──────────┬───────────────────────────────────────────────┤
│ 片区管理  │ 客户列表（a-table）                              │
│ ▼ 所有区域│ 序号 所属区域 名称 联系人 电话 地址 会员类型       │
│  ├ 华东区 │      会员号 积分 操作                            │
│  ├ 华南区 │                                                 │
│  └ 未划分 │ 分页器（共N条 / 10条/页）                         │
└──────────┴───────────────────────────────────────────────┘
```

### 实现要点

1. **布局**：`a-row` + `a-col` 分为左（宽 180px 固定）、右两列；左右分别用 `a-card` 包裹。
2. **左侧片区列表**：
   - 调用 `/api/area/list` 获取片区数据
   - 用 `a-menu` 展示，根节点「所有区域」（area_id=0）
   - 点击节点设置 `selectedAreaId`，触发右侧列表刷新
3. **右侧列表**：
   - 表格列：序号（行内 `(page-1)*pageSize + index + 1`）、所属区域、名称、联系人、电话、地址、会员类型（a-tag）、会员号、积分、操作
   - 会员类型 Tag 颜色：含"金"=gold、含"银"=blue、含"普通"=default、其他=green
   - 操作列三点下拉菜单（修改 / 充值 / 删除），删除用 `Modal.confirm()`
   - 底部 `a-pagination`，绑定 `current` + `pageSize`，变化时重新拉取数据
4. **新建/修改弹窗**（`a-modal`，宽度 600px）：
   - 分两组：**基本信息**（名称必填、联系人、联系电话、联系地址、所属区域下拉、所属业务员）
   - **会员信息**（会员类型下拉、会员号、会员生日 `a-date-picker`）
   - 区域数据、会员类型数据在组件 `onMounted` 时加载缓存
5. **充值弹窗**（独立 `a-modal`）：
   - 显示当前余额（只读）
   - 输入充值金额（`a-input-number`，min=0.01，precision=2，必填）
   - 确认后调用 `/api/customer/recharge`，成功后刷新列表
6. 在 `Home.vue` 的 `COMPONENT_MAP` 中注册：`'/system/customer': CustomerManage`

# BUG和优化点：
1）UI优化：每行末尾三点图标，下拉菜单「修改」「充值」「删除」，从`@ant-design/icons-vue`找合适的icon放在文字左侧。

3）交付优化：创建新客户和编辑客户对话框中，[客户名称]和[联系电话]设置为必填字段；任何一个必填字段为空，确认按钮不可点击。
