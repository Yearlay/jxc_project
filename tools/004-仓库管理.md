# 仓库管理

**功能**：实现三级菜单页面
- 系统维护 → 基础信息设置 → 仓库（菜单路径：`/system/basic/warehouse`）

---

# 输入

## UI 风格参考
效果图参考：`design/效果图/基础信息设置_片区管理.png`（仓库与片区管理页面风格完全一致）

- 顶部：「仓库信息」标签 + 搜索框（请输入仓库名称）+ 「查询」按钮 + 「创建新仓库」按钮（`PlusOutlined` 图标）
- 中间：仓库列表表格，无分页（数量有限，全量展示）
- 操作：每行末尾三点图标（`EllipsisOutlined`），下拉菜单含「修改」（`EditOutlined`）「删除」（`DeleteOutlined`，红色）
- 删除用 `Modal.confirm()` 二次确认

## 字段说明

| 字段       | 类型          | 说明                     |
|------------|---------------|--------------------------|
| id         | SERIAL PK     | 主键                     |
| name       | VARCHAR(100)  | 仓库名称（必填，唯一）   |
| address    | VARCHAR(200)  | 仓库地址                 |
| remark     | VARCHAR(200)  | 备注                     |
| created_at | TIMESTAMP     | 创建时间                 |

> 不需要 `status` 启用/禁用字段。

## 业务说明

- 列表支持按名称关键词搜索（ILIKE 任意匹配）。
- 全量展示，不分页。
- 创建新仓库：弹窗填写名称（必填）、地址、备注。
- 修改仓库：与创建使用同一弹窗，回填已有数据。
- 删除：`Modal.confirm()` 二次确认，删除后刷新列表。
- 仓库名称唯一，重复时后端返回错误提示。

## 接口设计

基础路径：`/api/warehouse`

| 方法   | 路径                       | 说明                             |
|--------|----------------------------|----------------------------------|
| GET    | /api/warehouse/list        | 仓库列表（支持 name 关键词搜索） |
| POST   | /api/warehouse/add         | 新建仓库                         |
| PUT    | /api/warehouse/update      | 修改仓库                         |
| DELETE | /api/warehouse/delete      | 删除仓库                         |

### list 接口返回格式
```json
{
  "code": 200,
  "data": [
    { "id": 1, "name": "北京总仓", "address": "北京市朝阳区XX路1号", "remark": "" },
    { "id": 2, "name": "上海分仓", "address": "上海市浦东新区XX路2号", "remark": "" }
  ],
  "msg": "success"
}
```

# 实现

## step 1: 在 `pg` 目录下，创建仓库表结构并初始化数据

文件命名：`07_create_warehouse_table.sql`

1. 创建 `biz_warehouse` 表，字段及约束参考【字段说明】，添加字段注释。
2. 初始化 3~5 条示例数据（覆盖不同地区仓库）。
3. 将新文件加入 `00_reinit_db.sh` 的执行序列（step7）。
4. 执行验证：
   ```
   psql -U onepiece -d jxc -h 192.168.0.116 -p 5432 -W -f 07_create_warehouse_table.sql
   ```

## step 2: 后端 `jxcServer` 工程，创建仓库管理 API

1. 新建 `server/jxcServer/api/warehouse.py`，注册蓝图 `warehouse_bp`，前缀 `/api/warehouse`，实现 4 个接口。
2. `list` 接口：支持 `name`（ILIKE 匹配），返回全量数组（无分页）。
3. `add` 接口：name 必填，name 唯一（重复时返回 400 + 友好提示）。
4. `update` 接口：id + name 必填，name 唯一校验。
5. `delete` 接口：直接删除，无关联检查（预留）。
6. 在 `app.py` 中导入并注册 `warehouse_bp`。
7. 统一返回格式 `{"code": 200, "msg": "...", "data": ...}`。
8. 接口完成后使用 `curl` 验证全部 4 个接口。

## step 3: 前端 `jxcClient` 实现「仓库」页面

文件：`src/views/WarehouseManage.vue`，新建 API 文件 `src/api/warehouse.js`

### 整体布局（参考效果图，与片区管理页面风格一致）
```
┌────────────────────────────────────────────────────────┐
│ 仓库信息  [搜索框 请输入仓库名称]  [查询]    [创建新仓库] │
├────────────────────────────────────────────────────────┤
│ 仓库信息                                      操作      │
│ 北京总仓                                       ···      │
│ 上海分仓                                       ···      │
│ 广州分仓                                       ···      │
└────────────────────────────────────────────────────────┘
```

### 实现要点

1. 页面整体参考 `AreaManage.vue` / `MemberTypeManage.vue` 风格。
2. **顶部工具栏**：左侧搜索框（`@pressEnter` + `@change` 清空时自动刷新），右侧「创建新仓库」按钮（`PlusOutlined`）。
3. **列表**：`a-table`，列为「仓库信息（name）」「地址」「备注」「操作」，`pagination=false`。
4. **操作列**：`EllipsisOutlined` 三点触发 `a-dropdown`，下拉菜单「修改」（`EditOutlined`）、「删除」（`DeleteOutlined`，红色），删除用 `Modal.confirm()`。
5. **新建/修改弹窗**（`a-modal`）：
   - 表单字段：仓库名称（必填）、仓库地址、备注
   - 取消时 `clearValidate()`，确认时 `validate()` 后提交
6. 在 `Home.vue` 的 `COMPONENT_MAP` 中注册：`'/system/basic/warehouse': WarehouseManage`
