# 片区管理 + 会员类型

**功能**：实现两个三级菜单页面
- 系统维护 → 基础信息设置 → 片区信息
- 系统维护 → 基础信息设置 → 会员类型

---

# 输入

## UI 风格参考
效果图：`design/效果图/基础信息设置_片区管理.png`
- 整体为带搜索栏的列表页，顶部搜索区 + 「新建」按钮，下方为数据列表表格。
- 列表每行右侧有三点操作图标（`EllipsisOutlined`），下拉菜单含「编辑」「删除」。
- 删除需 `a-popconfirm` 二次确认。

## 片区管理

### 字段说明

| 字段       | 类型         | 说明                          |
|------------|--------------|-------------------------------|
| id         | SERIAL PK    | 主键                          |
| name       | VARCHAR(50)  | 片区名称，唯一                |
| remark     | VARCHAR(200) | 备注（可为空）                |
| is_default | SMALLINT     | 是否默认：1=是（未划分）0=否  |
| status     | SMALLINT     | 状态：1=启用 0=禁用           |
| created_at | TIMESTAMP    | 创建时间                      |

### 业务说明
- 列表支持按名称搜索过滤。
- 可新建片区（输入名称 + 备注）、编辑、删除。
- 【未划分】为系统默认片区（`is_default=1`），**三点操作图标隐藏**，不允许编辑和删除。
- 删除前需检查是否有关联数据（预留，当前可直接删除）。

### 接口设计

基础路径：`/api/area`

| 方法   | 路径             | 说明             |
|--------|------------------|------------------|
| GET    | /api/area/list   | 片区列表（支持 name 关键词搜索） |
| POST   | /api/area/add    | 新建片区         |
| PUT    | /api/area/update | 编辑片区         |
| DELETE | /api/area/delete | 删除片区         |

## 会员类型

### 字段说明

| 字段         | 类型          | 说明                    |
|--------------|---------------|-------------------------|
| id           | SERIAL PK     | 主键                    |
| name         | VARCHAR(50)   | 会员类型名称，唯一      |
| discount     | NUMERIC(4,2)  | 折扣率（如 0.88 = 88折）|
| remark       | VARCHAR(200)  | 备注（可为空）          |
| status       | SMALLINT      | 状态：1=启用 0=禁用     |
| created_at   | TIMESTAMP     | 创建时间                |

### 业务说明
- 列表支持按名称搜索过滤。
- 可新建会员类型（输入名称 + 折扣率 + 备注）、编辑、删除。
- 折扣率范围：0.01 ~ 1.00，页面显示为百分比形式（如 88%）。
- 删除前需检查是否有关联顾客（预留，当前可直接删除）。

### 接口设计

基础路径：`/api/member-type`

| 方法   | 路径                   | 说明             |
|--------|------------------------|------------------|
| GET    | /api/member-type/list  | 会员类型列表（支持 name 关键词搜索） |
| POST   | /api/member-type/add   | 新建会员类型     |
| PUT    | /api/member-type/update| 编辑会员类型     |
| DELETE | /api/member-type/delete| 删除会员类型     |

# 实现

## step 1: 在 `pg` 目录下，创建表结构并初始化数据

文件命名：`05_create_area_membertype_tables.sql`

1. 创建 `biz_area`（片区信息表），添加字段注释。
2. 创建 `biz_member_type`（会员类型表），添加字段注释。
3. 初始化数据：
   - 插入默认片区【未划分】（`is_default=1`）。
   - 插入常见会员类型示例：普通会员（1.00）、银卡（0.95）、金卡（0.88）、钻石（0.80）。
4. 将新文件加入 `00_reinit_db.sh` 的执行序列。
5. 执行验证：
   ```
   psql -U onepiece -d jxc -h 192.168.0.116 -p 5432 -W -f 05_create_area_membertype_tables.sql
   ```

## step 2: 后端 `jxcServer` 工程，创建相关 API

1. 新建 `server/jxcServer/api/area.py`，注册蓝图 `area_bp`，前缀 `/api/area`，实现 4 个接口。
2. 新建 `server/jxcServer/api/member_type.py`，注册蓝图 `member_type_bp`，前缀 `/api/member-type`，实现 4 个接口。
3. 在 `app.py` 中注册两个蓝图。
4. 统一返回格式 `{"code": 200, "msg": "...", "data": ...}`。
5. 删除默认片区【未划分】时，后端拒绝并返回错误提示。
6. 接口完成后，使用 `curl` 逐一验证。

## step 3: 前端 `jxcClient` 实现「片区信息」页面

文件：`src/views/AreaManage.vue`

1. 布局：顶部搜索栏 + 「新建片区」按钮，下方 `a-table` 列表。
2. 表格列：片区名称、备注、状态、操作（三点菜单：编辑/删除）。
3. 【未划分】行（`is_default=1`）隐藏三点操作图标。
4. 新建/编辑：弹出 `a-modal`，包含名称（必填）、备注（可选）字段。
5. 删除：`a-popconfirm` 二次确认。
6. 在 `Home.vue` 的 `COMPONENT_MAP` 中注册路径 `/system/basic/area`。

## step 4: 前端 `jxcClient` 实现「会员类型」页面

文件：`src/views/MemberTypeManage.vue`

1. 布局：顶部搜索栏 + 「新建会员类型」按钮，下方 `a-table` 列表。
2. 表格列：类型名称、折扣率（显示为 xx%）、备注、状态、操作（三点菜单：编辑/删除）。
3. 新建/编辑：弹出 `a-modal`，包含名称（必填）、折扣率（必填，0.01~1.00）、备注（可选）字段。
4. 删除：`a-popconfirm` 二次确认。
5. 在 `Home.vue` 的 `COMPONENT_MAP` 中注册路径 `/system/basic/member-type`。
