# 商品列表

**功能**：实现二级菜单页面
- 商品管理 → 商品列表（菜单 id=23，path=/goods/list）

---

# 输入

### 商品（biz_goods）
1）字段说明：`序号`、`编码`、`其他编码`、`名称`、`分类`、`采购价`、`销售价`、`会员价`、`批发价`、`单位`、`生产厂家`、`当前库存`、`库存下限`、`保质期`、`参与积分`(是/否)、`会员折扣`(是/否)
2）业务说明：按名称/编号查询、按分类查询、新建商品、编辑、删除

### 商品库存（biz_goods_stock）
1）字段说明：`存放仓库`、`规格`、`批号`、`尺码`、`颜色`、`生产日期`、`有效日期`、`商品名称`、`当前库存`
2）业务说明：增加库存、库存盘点、仓库调拨

---

# 实现

## step 1: 在 `pg` 目录下，新建商品相关表结构 SQL

新建文件：`12_create_goods_tables.sql`

### 表一：biz_goods（商品表）

字段设计：
- `id SERIAL PRIMARY KEY`
- `code VARCHAR(50) NOT NULL UNIQUE`：商品编码，唯一
- `other_code VARCHAR(50) NOT NULL DEFAULT ''`：其他编码
- `name VARCHAR(100) NOT NULL`：商品名称
- `category_id INTEGER NOT NULL DEFAULT 0`：关联 `goods_category.id`，0 表示未分类
- `purchase_price NUMERIC(10,2) NOT NULL DEFAULT 0.00`：采购价（成本价）
- `sale_price NUMERIC(10,2) NOT NULL DEFAULT 0.00`：销售价（普通顾客价）
- `member_price NUMERIC(10,2) NOT NULL DEFAULT 0.00`：会员价（配合折扣率使用）
- `wholesale_price NUMERIC(10,2) NOT NULL DEFAULT 0.00`：批发价
- `unit_id INTEGER NOT NULL DEFAULT 0`：关联 `biz_unit.id`
- `manufacturer_id INTEGER NOT NULL DEFAULT 0`：关联 `biz_manufacturer.id`
- `stock_min INTEGER NOT NULL DEFAULT 0`：库存下限（预警值）
- `shelf_life INTEGER NOT NULL DEFAULT 0`：保质期，单位：天；0 表示不限
- `enable_points SMALLINT NOT NULL DEFAULT 0`：参与积分，1=是，0=否
- `enable_discount SMALLINT NOT NULL DEFAULT 0`：参与会员折扣，1=是，0=否
- `created_at TIMESTAMP NOT NULL DEFAULT NOW()`

约束：
- `code` 唯一
- 不设计状态列（启用/禁用）

备注：
- `当前库存` 是 `biz_goods_stock` 中同 `goods_id` 所有记录 `quantity` 之和，由后端聚合计算，不存在商品表中
- 若当前库存 > 0，则禁止删除该商品
- 当前库存 <= 库存下限时，前端商品列表该行显示红色字体

### 表二：biz_goods_stock（商品库存表）

字段设计：
- `id SERIAL PRIMARY KEY`
- `goods_id INTEGER NOT NULL`：关联 `biz_goods.id`
- `warehouse_id INTEGER NOT NULL`：关联 `biz_warehouse.id`（存放仓库）
- `spec VARCHAR(100) NOT NULL DEFAULT ''`：规格
- `batch_no VARCHAR(100) NOT NULL DEFAULT ''`：批号
- `size VARCHAR(50) NOT NULL DEFAULT ''`：尺码
- `color VARCHAR(50) NOT NULL DEFAULT ''`：颜色
- `produce_date DATE`：生产日期，可为空
- `quantity INTEGER NOT NULL DEFAULT 0`：当前库存（整数）
- `created_at TIMESTAMP NOT NULL DEFAULT NOW()`

备注：
- `有效日期` = `produce_date` + 商品的 `shelf_life` 天数，由后端或前端计算，不存库
- 同步更新 `00_reinit_db.sh`，追加 step 12 执行 `12_create_goods_tables.sql`

## step 2: 后端 `jxcServer` 工程，创建商品相关 API

新建文件：`api/goods.py`

Blueprint：`goods_bp`，在 `app.py` 中注册，`url_prefix='/api'`

统一返回格式：`{ code, msg, data }`（沿用现有 `_ok` / `_err` 工具函数）

### 商品接口

#### 1. 商品列表（分页+筛选）
- `GET /api/goods/list`
- Query：`page`（默认1）、`page_size`（默认20）、`keyword`（名称/编码模糊）、`category_id`（分类筛选，0=全部）
- 返回：分页数据 `{ total, list }`，每条记录含 `stock_total`（当前库存聚合值）、`category_name`、`unit_name`、`manufacturer_name`

#### 2. 新增商品
- `POST /api/goods/add`
- Body：`{ code, other_code, name, category_id, purchase_price, sale_price, member_price, wholesale_price, unit_id, manufacturer_id, stock_min, shelf_life, enable_points, enable_discount }`
- 规则：`code`、`name` 不能为空；`code` 不允许重复

#### 3. 修改商品
- `PUT /api/goods/update`
- Body：`{ id, code, other_code, name, category_id, purchase_price, sale_price, member_price, wholesale_price, unit_id, manufacturer_id, stock_min, shelf_life, enable_points, enable_discount }`
- 规则：`id` 必填；`code` 不允许与其他商品重复

#### 4. 删除商品
- `DELETE /api/goods/delete`
- Body：`{ id }`
- 规则：当前库存 > 0 时，禁止删除，返回错误提示

### 商品库存接口

#### 5. 库存清单（按商品查询）
- `GET /api/goods/stock/list?goods_id=xxx`
- 返回该商品所有库存记录，含 `warehouse_name`、`expire_date`（后端计算：produce_date + shelf_life 天）

#### 6. 增加库存
- `POST /api/goods/stock/add`
- Body：`{ goods_id, warehouse_id, spec, batch_no, size, color, produce_date, quantity }`
- 规则：`goods_id`、`warehouse_id`、`quantity` 必填；`quantity` > 0

#### 7. 库存盘点（编辑单条库存记录）
- `PUT /api/goods/stock/update`
- Body：`{ id, warehouse_id, spec, batch_no, size, color, produce_date, quantity }`
- 规则：`id` 必填；`quantity` >= 0（允许盘点为 0）

#### 8. 仓库调拨
- `POST /api/goods/stock/transfer`
- Body：`{ goods_id, from_warehouse_id, to_warehouse_id, quantity }`
- 规则：
  - `from_warehouse_id` 与 `to_warehouse_id` 不能相同
  - `quantity` > 0 且不超过源仓库该商品的库存总量
  - 调拨逻辑：在源仓库库存中扣减 quantity（可减到 0），在目标仓库新增一条 stock 记录

## step 3: 前端 `jxcClient` 实现 "商品列表" 菜单页面

新建文件：`src/views/GoodsManage.vue`
新建文件：`src/api/goods.js`（封装上述所有接口）

在 `src/views/Home.vue` 的 `COMPONENT_MAP` 中追加：
```js
import GoodsManage from './GoodsManage.vue'
// COMPONENT_MAP 中添加：
'/goods/list': GoodsManage,
```

### 页面整体布局

参考 `GoodsCategoryManage.vue` + `CustomerManage.vue` 的风格，整体三区域：

```
┌─────────────────────────────────────────────────────┐
│ [搜索框：名称/编码] [+ 新建商品] 按钮                    │
├───────────────┬─────────────────────────────────────┤
│               │                                     │
│  商品分类树     │  商品列表（a-table，分页）             │
│  (a-tree)     │  每行右侧：三点操作菜单                │
│               │                                     │
└───────────────┴─────────────────────────────────────┘
```

### 左侧：商品分类树
- 调用 `GET /api/goods-category/tree` 获取分类树
- 使用 `a-tree`，顶部加"所有商品"虚拟根节点（category_id=0 表示不过滤）
- 点击节点，以 `category_id` 筛选右侧商品列表，并重置到第 1 页

### 右侧上部：工具栏
- 搜索框：按名称/编码模糊查询（输入防抖 500ms 后触发）
- 新建商品按钮（`PlusOutlined` 图标，type="primary"）

### 右侧：商品列表（a-table）

展示列：序号、编码、名称、分类、采购价、销售价、会员价、批发价、当前库存、单位、保质期（天）、生产厂家、操作

列规则：
- 当前库存 <= 库存下限时，"当前库存"单元格文字显示红色

操作列（每行右侧三点菜单，始终显示）：

| 菜单项   | 显示条件              |
| -------- | --------------------- |
| 编辑     | 始终显示              |
| 删除     | 当前库存 = 0 时显示   |
| 库存清单 | 始终显示              |
| 增加库存 | 始终显示              |
| 库存盘点 | 当前库存 > 0 时显示   |
| 仓库调拨 | 当前库存 > 0 时显示   |

分页：默认 page_size=20，底部 `a-pagination`

### 对话框设计

#### 新建/编辑对话框
- 宽度：640px，两列表单布局
- 字段：
  - 商品编码（必填）、其他编码
  - 商品名称（必填）、分类（a-tree-select，加载分类树）
  - 采购价（数字输入）、销售价（数字输入）
  - 会员价（数字输入）、批发价（数字输入）
  - 单位（a-select，加载 biz_unit 列表）、生产厂家（a-select，加载 biz_manufacturer 列表）
  - 库存下限（数字输入）、保质期（数字输入，单位：天）
  - 参与积分（a-switch，是/否）、会员折扣（a-switch，是/否）
- 编辑模式：当前库存只读展示，不可修改

#### 删除对话框
- `Modal.confirm`，内容："确认删除商品《xxx》？删除后不可恢复。"
- 调用 `DELETE /api/goods/delete`，若有库存则后端拦截并 toast 提示

#### 库存清单对话框
- 标题：`《商品名称》— 库存清单`
- `a-table`，列：仓库、规格、批号、尺码、颜色、生产日期、有效日期、当前库存
- 调用 `GET /api/goods/stock/list?goods_id=xxx`

#### 增加库存对话框
- 字段：
  - 存放仓库（a-select，加载全部仓库，必填）
  - 规格、批号、尺码、颜色（文本输入，选填）
  - 生产日期（a-date-picker，选填）
  - 增加数量（正整数，必填，> 0）
- 调用 `POST /api/goods/stock/add`

#### 库存盘点对话框
- Step 1：下拉选择仓库（过滤出该商品有库存记录 quantity>0 的仓库）
- Step 2：选仓库后，展示该仓库下该商品的库存记录（a-table 可编辑行），列：规格、批号、尺码、颜色、生产日期、数量
- 点"保存"：循环调用 `PUT /api/goods/stock/update`

#### 仓库调拨对话框
- 字段：
  - 源仓库（a-select，过滤出该商品有库存 quantity>0 的仓库，必填）
  - 目标仓库（a-select，加载全部仓库，必填；不能与源仓库相同）
  - 源仓库库存（只读，选择源仓库后联动显示该仓库的库存总量）
  - 调拨数量（正整数，必填；需 <= 源仓库库存数量）
- 按钮：调拨（primary）/ 取消
- 调用 `POST /api/goods/stock/transfer`
